# LinkNote v2

LinkNote 是一個可部署到 GitHub Pages 的純前端離線筆記 App，支援：

- Markdown 編輯與預覽
- `[[雙向連結]]`、反向連結與標籤
- 知識圖譜
- 由 Markdown 標題、縮排清單與雙向連結自動產生的心智圖
- 可編輯的策略地圖（四構面、KPI、因果箭頭、SVG 匯出）
- IndexedDB 儲存與舊版 `localStorage` 自動搬移
- JSON 合併匯入，不整批覆蓋現有筆記
- 垃圾桶、刪除後復原與永久刪除
- 每篇筆記最多 30 個版本紀錄
- 完整 JSON 備份、Markdown 匯出與備份提醒
- PWA／iPhone 加入主畫面與離線快取

## 更新既有 GitHub 專案

1. 先將舊版專案下載一份作為程式備份。
2. 用本資料夾的 `index.html`、`styles.css`、`app.js`、`sw.js`、`manifest.webmanifest`、`README.md` 與 `icons` 覆蓋儲存庫中的同名檔案。
3. Commit 並 Push 到 `main`。
4. GitHub Pages 若已啟用，等待重新部署後開啟網站。
5. 首次開啟新版時，舊版 `linknote.notes.v1` 筆記會自動搬移至 IndexedDB。

## 合併匯入規則

- 不會先清空現有筆記。
- 以相同 ID 或同名標題判斷為同一篇。
- 內容完全相同時不重複加入。
- 其中一份內容包含另一份時保留較完整內容。
- 內容不同時，在原筆記下方加入「匯入合併內容」區段。
- 策略地圖目標依構面與名稱合併，因果箭頭也會一起合併。

## 心智圖格式

心智圖直接讀取目前筆記內容：

```markdown
# 研究主題

## 文獻回顧
- 生成式 AI
- 知識圖譜
  - [[相關論文]]

## 系統實作
- 前端
- 資料庫
```

## 策略地圖操作

1. 開啟一篇筆記，按右上角 `⇧`。
2. 選擇構面、輸入目標並按「新增目標」。
3. 點選目標後可按「編輯所選」加入 KPI。
4. 按「建立連線」，依序點選起點與終點。
5. 四個構面可以重新命名，例如改為教育專案常用的成果、學習者、教學流程、能力建設。

## 資料安全限制

IndexedDB 與持久儲存權限能降低瀏覽器因空間不足而自動清理資料的機率，但使用者若在 Safari 或 Chrome 設定中主動清除該網站資料，純前端 App 的本機資料仍會被刪除。請定期使用「匯出完整備份」將 JSON 檔案存到 iCloud Drive、Google Drive 或電腦。
