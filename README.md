# LinkNote v2.2

LinkNote 是一個可部署到 GitHub Pages 的純前端離線筆記 App，支援 Markdown、雙向連結與多種筆記內視覺化。

## 主要功能

- Markdown 編輯與預覽
- `[[雙向連結]]`、反向連結與標籤
- 目前筆記內的 Knowledge Graph
- 由 Markdown 標題與縮排清單產生的心智圖
- **流程圖：步驟、判斷、分支與循環**
- **樹狀圖：階層分類、收合與展開**
- **引導式策略地圖：目標、KPI、構面、因果關係與範例模板**
- IndexedDB 儲存與舊版 `localStorage` 自動搬移
- JSON 合併匯入，不整批覆蓋現有筆記
- 垃圾桶、刪除復原與每篇最多 30 個版本紀錄
- SVG、Markdown 與完整 JSON 備份匯出
- PWA、iPhone 加入主畫面與離線快取

## v2.2 新增

### 流程圖

在筆記中加入 `flow` 程式區塊：

````markdown
```flow
開始((開始)) -> 蒐集資料[蒐集資料]
蒐集資料 -> 判斷{資料完整？}
判斷 ->|是| 分析[分析資料]
判斷 ->|否| 蒐集資料
分析 -> 完成((完成))
```
````

支援的形狀：

- `代號[文字]`：一般步驟
- `代號{文字}`：判斷菱形
- `代號((文字))`：開始或結束
- `A -> B`：建立箭頭
- `A ->|是| B`：建立有標籤的箭頭
- `A -> B : 促進`：另一種關係標籤寫法

按上方 `↬` 開啟流程圖，可匯出 SVG。

### 樹狀圖

可直接使用目前筆記的 Markdown 標題與縮排條列，也能使用專用 `tree` 區塊：

````markdown
```tree
多語言學習
  中文
    發音
    文法
  韓語
    字母
    敬語
```
````

按上方 `♧` 開啟樹狀圖。點擊有子節點的方塊可收合或展開，也能一鍵「收合到兩層」或「全部展開」。

### 策略地圖加強

策略地圖改為引導式操作：

1. 選擇構面，輸入策略目標及 KPI。
2. 用起點、關係、終點三個欄位建立因果箭頭。
3. 從連線清單檢查或移除關係。
4. 點擊地圖節點後，在表單中修改目標、KPI 或所屬構面。
5. 可按「載入範例」直接建立一份完整示範地圖。
6. 可清空地圖；清空前會留下版本快照，仍可從版本紀錄復原。

預設由下往上閱讀：

- 學習與成長：人才、技術、資料與資源
- 內部流程：需要做好的關鍵工作
- 學習者／利害關係人：使用者應感受到的價值
- 成果／財務：最後希望達成的成果

每條箭頭都可以設定「促進、支援、提升、帶動」等關係文字。

## 筆記內 Knowledge Graph

一般 Markdown 標題與條列會成為概念節點。要明確指定概念關係，可加入：

````markdown
```knowledge
生成式 AI | 提供 | 即時回饋
即時回饋 | 提升 | 學習成效
學習分析 -> 支援 -> 個人化學習
知識圖譜 → 概念連結
```
````

支援：

- `主詞 | 關係 | 受詞`
- `主詞 -> 關係 -> 受詞`
- `主詞 → 受詞`
- `主詞 --關係--> 受詞`

按上方 `◎` 開啟目前筆記的 Knowledge Graph。

## 心智圖

心智圖會讀取目前筆記的 Markdown 標題、縮排清單與 `[[連結]]`。按上方 `⌘` 開啟。

## 更新既有 GitHub 專案

1. 先從舊版 LinkNote 匯出完整 JSON 備份。
2. 將本資料夾檔案覆蓋 GitHub 儲存庫中的同名檔案。
3. Commit 並 Push 到 `main`。
4. 等待 GitHub Pages 重新部署。
5. 首次開啟後重新整理一次；iPhone 主畫面版若仍是舊畫面，完全關閉後再開啟。

新版已更新 Service Worker 快取名稱，一般不需要清除網站資料。

## 資料安全限制

IndexedDB 與持久儲存權限能降低瀏覽器自動清理資料的機率，但使用者主動清除 Safari 或 Chrome 網站資料時，本機筆記仍可能被刪除。請定期將完整 JSON 備份保存到 iCloud Drive、Google Drive 或電腦。
