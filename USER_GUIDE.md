# LinkNote 使用說明書

本說明適用於 LinkNote v2.3。App 內也可按左側的「？ 使用說明與範例」查看相同概念，並一鍵建立測試筆記。

## 1. 最快開始方式

1. 按「＋ 新增筆記」。
2. 輸入筆記標題與內容。
3. 依照用途使用一般 Markdown 或特殊圖表區塊。
4. 按上方圖示開啟對應視覺化。
5. 需要保存圖表時按「匯出 SVG」；需要保存全部筆記時到設定匯出 JSON。

## 2. 上方圖示對照

| 圖示 | 功能 | 最適合的內容 |
|---|---|---|
| `◎` | Knowledge Graph | 概念之間的因果、影響、包含、支援等關係 |
| `⌘` | 心智圖 | 從中心主題向外展開的想法與文章大綱 |
| `↬` | 流程圖 | 有先後順序、判斷、分支與循環的流程 |
| `♧` | 樹狀圖 | 分類、組織、章節、上下層級 |
| `⇧` | 策略地圖 | 能力、流程、使用者價值與最終成果的因果鏈 |
| `♲` | 垃圾桶 | 還原或永久刪除筆記 |
| `⚙` | 設定與備份 | JSON 備份、合併匯入、持久儲存與安裝說明 |

## 3. Knowledge Graph

### 適合情況

- 整理研究概念與理論關係
- 表示人物、事件、地點的關係
- 描述系統元件如何互相影響
- 把因果關係明確寫出來

### 格式

````markdown
```knowledge
主詞 | 關係 | 受詞
睡眠品質 | 提升 | 注意力
注意力 | 影響 | 學習成效
運動 -> 改善 -> 睡眠品質
``` 
````

也可以寫：

```text
主詞 → 受詞
主詞 --關係--> 受詞
```

### 操作

1. 在目前筆記加入 `knowledge` 區塊。
2. 每一條關係獨立一行。
3. 按上方 `◎`。
4. 檢查箭頭方向與關係文字。
5. 按「匯出 SVG」保存。

## 4. 心智圖

### 適合情況

- 腦力激盪
- 文章或報告大綱
- 課程知識整理
- 專案工作拆解

### 格式

心智圖不需要特殊程式區塊，直接使用標題與縮排清單：

```markdown
# 博物館展覽企劃

## 展覽內容

- 城市記憶
  - 市場
  - 學校
  - 交通

## 觀眾體驗

- 語音導覽
- 互動地圖
```

### 操作

1. 以 `#`、`##`、`###` 建立大層級。
2. 以縮排條列建立子層級。
3. 按上方 `⌘`。
4. 使用 `－`、`重設`、`＋` 調整大小。

## 5. 流程圖

### 適合情況

- 行政申請與審核
- App 使用流程
- 研究方法程序
- 問題排除步驟
- 具有「是／否」判斷的工作

### 格式

````markdown
```flow
開始((開始)) -> 申請[提交申請]
申請 -> 審核{資料完整？}
審核 ->|否| 補件[通知補件]
補件 -> 申請
審核 ->|是| 通過((審核通過))
```
````

### 形狀

| 寫法 | 顯示 |
|---|---|
| `A[一般步驟]` | 長方形 |
| `A{判斷問題？}` | 菱形 |
| `A((開始或結束))` | 圓角終端 |
| `A -> B` | 一般箭頭 |
| `A ->|是| B` | 帶文字的箭頭 |

### 注意事項

- 同一個節點請維持相同代號。
- 每條流程獨立一行。
- 若要回到前一步，可再次指向舊節點。

## 6. 樹狀圖

### 適合情況

- 生物分類
- 組織架構
- 課程章節
- 檔案目錄
- 產品類別

### 格式

````markdown
```tree
生物
  動物界
    脊椎動物
      哺乳類
      鳥類
  植物界
    種子植物
    非種子植物
```
````

### 注意事項

- 建議每一層使用兩個空白。
- 不要混用 Tab 與空白。
- 點擊有子節點的方塊可以收合或展開。

## 7. 策略地圖

策略地圖不是單純的分類圖，而是回答：**我們要培養什麼能力，做好哪些流程，為使用者創造什麼價值，最後得到什麼成果？**

### 四個構面

1. **成果／財務**：最後希望達成的成果。
2. **學習者／利害關係人**：使用者應感受到的價值。
3. **內部流程**：必須做好的關鍵工作。
4. **學習與成長**：所需的人才、技術、資料與資源。

### 建議建立順序

1. 先定義最上層成果，例如「提高回購率」。
2. 思考使用者必須感受到什麼，例如「縮短等待時間」。
3. 思考內部流程必須怎麼改善，例如「縮短出餐流程」。
4. 思考需要培養什麼能力，例如「員工數位工具訓練」。
5. 建立箭頭時由下往上連：

```text
員工數位工具訓練
        ↓ 支援
縮短點餐與出餐流程
        ↓ 降低
顧客等待時間
        ↓ 促進
顧客回購率
```

### KPI 寫法

好的 KPI 應包含可衡量數字與期限：

- 不清楚：提升顧客滿意度
- 較清楚：六個月內將滿意度提升至 4.5／5
- 不清楚：改善出餐速度
- 較清楚：三個月內將平均出餐時間降至 8 分鐘

### App 操作

1. 按上方 `⇧`。
2. 選構面，輸入策略目標及 KPI。
3. 按「新增目標」。
4. 在「建立因果箭頭」選擇起點、關係與終點。
5. 點地圖中的目標可修改內容或構面。
6. 從連線清單移除錯誤關係。
7. 初次使用可按「載入範例」。

## 8. 雙向連結與標籤

### 雙向連結

```text
[[韓語學習]]
```

- 已存在：點擊後開啟該筆記。
- 不存在：點擊後建立新筆記。
- 被其他筆記連結時，會出現在「反向連結」。

### 標籤

```text
#研究 #韓語 #教學
```

點標籤可篩選相關筆記。

## 9. 資料安全

- 刪除筆記會先進垃圾桶。
- 每篇筆記最多保存 30 個版本。
- JSON 匯入會合併筆記，不會直接清空全部資料。
- 清除 Safari 或 Chrome 網站資料仍可能刪除 IndexedDB。
- 建議每 1～2 週匯出完整 JSON，保存到電腦、iCloud Drive 或 Google Drive。

## 10. 常見問題

### 按圖表按鈕沒有內容

確認目前開啟的筆記有正確格式。Knowledge Graph、流程圖和樹狀圖的程式區塊名稱必須分別是：

```text
knowledge
flow
tree
```

### 心智圖結構很亂

使用 Markdown 標題切分大段落，並保持條列縮排一致。

### 策略地圖沒有從文章自動出現

策略地圖是該篇筆記的獨立可編輯資料，不會直接解析文章中的「目標 A」。請在策略地圖視窗建立目標，或按「載入範例」。

### 更新 GitHub Pages 後仍是舊版

重新整理頁面；iPhone 主畫面版本則完全關閉後再開啟。新版已更新 Service Worker 快取名稱，一般不需要清除網站資料。
