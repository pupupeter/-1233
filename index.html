LinkNote 使用說明與範例

先判斷想表達的是「關係、發想、流程、分類」或「策略」，再選擇適合的圖。

第一次使用：四個步驟

  1. 新增筆記：按左側「＋ 新增筆記」，輸入標題與內容。
  2. 套用格式：依下方範例使用 Markdown、knowledge、flow 或 tree 區塊。
  3. 開啟圖表：按上方對應圖示；圖表只分析目前開啟的筆記。
  4. 保存成果：可匯出 SVG；筆記資料則定期匯出完整 JSON 備份。
不會覆蓋同名筆記；已存在的範例會略過。

我該使用哪一種圖？

Knowledge Graph概念之間有明確關係例如：降雨「增加」淹水風險。
心智圖從中心主題向外發想例如：研究題目的章節與想法。
流程圖工作有先後順序或判斷例如：申請、審核、補件、通過。
樹狀圖內容是分類與上下層級例如：生物分類、組織架構。
策略地圖目標之間有因果與成果例如：培訓員工 → 改善流程 → 提升滿意度。
◎ Knowledge Graph：概念關係圖

適合：研究概念、因果關係、人物與事件、系統元件關係。

寫法：每一行是一個「主詞｜關係｜受詞」。

```knowledge
咖啡因 | 提高 | 警覺性
睡眠不足 | 降低 | 專注力
運動 -> 改善 -> 睡眠品質
```

操作：儲存筆記後按上方「◎」。箭頭方向會從主詞指向受詞。

⌘ 心智圖：從主題展開想法

適合：腦力激盪、文章大綱、課程整理、專案規劃。

寫法：使用 Markdown 標題與縮排條列，不需要特殊程式區塊。

# 博物館展覽企劃
## 展覽內容
- 主題
  - 城市記憶
  - 日常生活
## 觀眾體驗
- 導覽
- 互動裝置

操作：按上方「⌘」，可放大、縮小及匯出 SVG。

↬ 流程圖：步驟、判斷與分支

適合：行政流程、使用者操作、研究程序、問題排除。

```flow
開始((開始)) -> 申請[提交申請]
申請 -> 審核{資料完整？}
審核 ->|否| 補件[通知補件]
補件 -> 申請
審核 ->|是| 通過((審核通過))
```
  • A[文字]：一般步驟
  • A{文字}：判斷
  • A((文字))：開始或結束
  • A ->|是| B：有標籤的分支
♧ 樹狀圖：分類與階層

適合：分類學、組織架構、檔案目錄、課程章節。

```tree

    哺乳類

```

每往下一層縮排兩個空白。開圖後可點擊節點收合或展開。

⇧ 策略地圖：把能力連到最終成果

適合：組織策略、個人學習計畫、產品發展與專案管理。

4. 成果／財務最後想達成什麼？
3. 學習者／利害關係人使用者要感受到什麼價值？
2. 內部流程哪些工作必須做好？
1. 學習與成長需要哪些能力、技術與資源？

建議順序：先寫最上層成果，再往下找必要條件；建立箭頭時則由下往上連。

完整例子：員工數位訓練 → 縮短點餐流程 → 降低顧客等待時間 → 提升回購率。

KPI 寫法：使用可衡量的數字與期限，例如「三個月內將平均等待時間降至 5 分鐘」。

常見問題

按圖表後是空白？
確認目前筆記有內容，並檢查程式區塊名稱是否正確：knowledge、flow、tree。
流程圖只有部分節點？
每一條流程請獨立一行，節點代號需保持一致，例如都使用「審核」。
樹狀圖層級不正確？
請使用空白縮排，建議每一層兩個空白，不要混用 Tab。
策略地圖沒有從文章自動生成？
策略地圖是獨立的可編輯資料，需要在策略地圖視窗新增目標，或按「載入範例」。
圖表與筆記是否會同步？
Knowledge Graph、心智圖、流程圖與樹狀圖會重新分析筆記；策略地圖則獨立保存在該篇筆記中。

垃圾桶
